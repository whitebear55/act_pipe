실제 로봇을 구동한 뒤, 실시간으로 각 관절의 pos값을 읽어와서 MuJoCo에게 보내고 mj_forward함수를 통해서 중력보상 토크를 계산
계산된 중력 보상 토크는 실제 로봇의 각 관절에서 출력하는 토크에서 빼서 각 관절에 가해지는 순수 외력 토크를 구하게 되고, 자코비안을 이용해서 End-effector에 가해지는 Wrench로 계산하게 돼

cf. 내가 실제로 겪은 문제 + 주의해야할 사항
<문제>
(1) 실제로 내가 시뮬레이션을 통해 로봇에 외력을 가하지 않은 상태에서 
tau_raw(실제 각 관절에서 출력하는 토크)값과 tau_bias(현재 자세를 유지하기 위한 중력 보상 토크)값을 측정해본 결과 두 값은 동일해서 제자리에 잘 위치해있었어. 이러한 이유는 로봇의 관절 각도에 맞는 현재 자세를 유지하기 위한 토크만 각 관절에서 내면 되므로 
그러나, 실제 Mobile Aloha로봇에 어드미턴스 제어 적용을 했을때의 문제는 초기위치에서의 tau_raw값과 tau_bias가 다른 문제가 발생(원인 미상)했고, 이를 위해서 초기 100프레임동안 오프셋을 측정하고 이를 보정하는 방식으로 설계를 진행했고, EE에 가해지는 wrench에 DeadBand를 걸어놔서 진동을 억제했어

<주의사항> 
(1) 실제로 진동을 억제하기 위해서는 각 로봇팔의 모터를 current base position모드가 아닌 position 모드로 수행해야함

(2) gin파일을 보면 ControllerConfig.fixed_base argument가 존재하는데, 이걸 true로 하면 현재 베이스의 좌표를 실시간으로 계산해서 어드미턴스 제어가 제대로 적용이 되지 않을 수 있음
따라서, 해당 argument를 False로 한다면 나에게 알려주거나 허락받기를 바람
